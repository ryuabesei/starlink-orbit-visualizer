Methodology

計算方法

このページは、Starlink Orbit Visualizer がTLEから衛星位置・高度・高度変化・熱圏密度プロキシをどのように計算しているかを、実装に対応する形で整理したものです。

可視化に戻る

1. 入力データ

入力はCelesTrakのStarlink TLEです。TLEは衛星名、1行目、2行目の3行1組として扱います。 

STARLINK-XXXX
1 NNNNNU ...
2 NNNNN ...

実装では各TLEを satellite.twoline2satrec(line1, line2) に渡し、SGP4で伝播できる satrec オブジェクトへ変換します。

NORAD番号satrec.satnum
TLE epochsatrec.jdsatepoch
平均運動satrec.no
軌道傾斜角satrec.inclo
BSTARsatrec.bstar

2. 衛星位置の計算

任意の時刻 t について、SGP4でTLE epochから衛星の位置と速度を伝播します。 

propagated = satellite.propagate(satrec, t)
r_eci = propagated.position   // km
v_eci = propagated.velocity   // km/s

この時点の位置はECI座標です。地球上の緯度・経度・高度や、Three.js上の地球固定表示に使うため、GMSTを計算して地球固定座標へ変換します。 

gmst = satellite.gstime(t)
r_ecf = satellite.eciToEcf(r_eci, gmst)
geodetic = satellite.eciToGeodetic(r_eci, gmst)

3. 高度・緯度・経度・速度

eciToGeodetic の結果から現在高度、緯度、経度を取り出します。緯度経度はラジアンから度へ変換します。 

altitude_km = geodetic.height
latitude_deg = degreesLat(geodetic.latitude)
longitude_deg = degreesLong(geodetic.longitude)

速度はECI速度ベクトルの大きさです。 

speed_km_s = sqrt(vx^2 + vy^2 + vz^2)

4. 3D地球へのプロット

地球半径をThree.js空間で 1 とし、実距離kmを縮尺変換します。地球平均半径は 6371 km としています。 

scale = 1 / 6371
x = r_ecf.x * scale
y = r_ecf.z * scale
z = -r_ecf.y * scale

y と z の入れ替えは、Three.jsの座標系と地球表示の向きを合わせるためです。

5. 高度レイヤー色分け

現在高度 altitude_km に応じて、衛星点と軌道線を色分けします。

< 400 km低高度
400-500 km主な低軌道帯
500-560 km標準的な運用帯
560 km+高めのシェル

6. 軌道線

軌道線は、中心時刻の前後50分、合計100分を96分割して、各時刻のSGP4位置を連続線として描いています。 

for i = 0..96:
    t_i = center_time - 50min + i * (100min / 96)
    point_i = positionFromSGP4(satrec, t_i)

7. 選択衛星の高度変化

衛星をクリックしたときの高度変化グラフは、現在時刻の前後90分を5分刻みで再計算しています。 

for minute = -90 to +90 step 5:
    t_i = selected_time + minute
    altitude_i = altitudeFromSGP4(satrec, t_i)

横軸は時間、縦軸は高度kmです。現在時刻は縦線と点で示しています。

8. 熱圏密度プロキシ

TLEだけから熱圏密度の絶対値を直接求めることはできません。このアプリでは、TLEに含まれる BSTAR を使い、相対的な大気抵抗・密度の proxy として表示しています。

実装では abs(BSTAR) の対数を取り、見やすい 0-100 の指数に正規化しています。 

bstar = satrec.bstar
magnitude = abs(bstar)
logValue = log10(magnitude)
proxy = clamp(((logValue + 6.5) / 3.5) * 100, 0, 100)

つまり、概念的には次のような見方です。 

|BSTAR| が小さい → proxy低い
|BSTAR| が大きい → proxy高い

ただしBSTARには、大気密度だけでなく、衛星の形状、姿勢、質量、抗力係数、軌道制御、TLEフィッティング誤差も混ざります。そのため、この値は絶対的な熱圏密度ではありません。

9. 熱圏密度プロキシ散布図

散布図は、各衛星について次の2つを使います。 

x = 現在高度 altitude_km
y = BSTAR由来 proxy

点の色は高度レイヤーと同じです。表示点数が多すぎる場合は、描画負荷を抑えるためにサンプリングして表示します。

10. 限界と今後の改善

現在の推定は、TLE 1時点のBSTARから作る相対指標です。より物理的に熱圏密度へ近づけるには、TLE時系列を使って軌道減衰を推定する必要があります。 

TLE時系列
→ 平均運動や半長軸の変化を計算
→ 軌道減衰率を推定
→ 仮定した Cd, A, m から密度を逆算
→ F10.7, Ap, Kp など太陽活動データで補正
→ NRLMSISE-00 などの大気モデルと比較

現在のページでは、プロダクト上の表示名として「熱圏密度プロキシ」または「BSTAR drag proxy」と呼ぶのが正確です。
